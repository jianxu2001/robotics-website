# SCR Robot 首页轻动画与宣传视频设计

## 目标

在不明显增加首页首屏加载成本的前提下，让 SCR Robot 首页更有动态感，并为 60 秒双语宣传片提供清晰入口。

已确认采用方案 A：三张真实工厂应用图柔和淡入淡出。完整宣传片不作为首屏背景，不自动下载。

## 首页首屏

- 保留现有标题、说明、询盘按钮和数据区，避免改变首页核心转化路径。
- 使用现有三张工厂应用图片作为背景序列。
- 图片每 4 秒交叉淡入淡出，一轮约 12 秒并持续循环。
- 仅使用 `opacity` 和 `transform` 实现轻微缩放及平移，不使用横向滑动。
- 文案、按钮、遮罩和工业网格保持稳定，避免内容跟随背景移动。
- 不引入第三方轮播或动画依赖。

## 加载与性能

- 第一张首屏图作为 LCP 候选图优先加载。
- 第一张图由服务端直接输出，确保 JavaScript 尚未运行时也能显示完整首屏。
- 第二、第三张图在页面可交互后按顺序挂载，不与第一张 LCP 图片竞争首屏带宽。
- 三张图片通过 Next.js `Image` 组件生成响应式尺寸和现代图片格式。
- 使用准确的 `sizes`，避免手机端下载桌面尺寸图片。
- 三张首屏图片的目标总传输体积为 300–500 KB；若现有源图无法达到目标，应先生成尺寸合理的优化源图。
- 动画使用 CSS，不增加轮播运行库和额外网络请求。
- 动画不得改变元素尺寸或位置布局，避免 CLS。

## 动画与可访问性

- 默认使用柔和交叉淡化，单次过渡约 1.2–1.8 秒。
- 使用 `prefers-reduced-motion: reduce` 时关闭轮播和缩放，仅显示第一张静态背景图。
- 背景图属于装饰内容，不重复向屏幕阅读器播报三张图片说明。
- 标题、按钮和正文对比度不因背景切换而下降。
- 页面在 JavaScript 未执行时仍需显示第一张背景图和全部首屏内容，后续图片不再切换。

## 完整宣传视频

- 在现有 “Watch the Robot in Action” 区域加入一个大幅宣传片入口。
- 初始状态只显示优化后的封面图、播放按钮、60 秒说明和简短文案。
- 访客点击播放按钮后才创建或激活视频播放器并请求视频文件。
- 视频不自动播放，初始设置 `preload="none"`，保留声音、播放进度和全屏控制。
- 原始 73.6 MB 文件不直接用于网页交付。实施阶段生成网页压缩版本，并保留原始文件作为制作母版。
- 视频区域加载失败时继续显示封面和联系入口，不影响首页其他内容。

## 组件边界

- 新建一个小型客户端组件负责在页面可交互后挂载后续图片、控制当前图片状态和定时切换。
- 图片淡化、缩放和平移全部由 CSS 完成，客户端逻辑仅改变活动状态。
- 首页页面继续作为服务端组件，静态文案、SEO 数据和主体结构不迁移到客户端。
- 宣传视频使用独立的延迟加载播放器组件，避免把视频交互逻辑耦合到首屏轮播。
- 动画样式放在现有全局样式中或组件级样式中，遵循当前项目的 Tailwind/CSS 写法。

## 验收标准

- 首页首屏可看到三张图片按顺序柔和切换，无横向滑动或明显闪烁。
- 文案、按钮和数据卡在切换期间保持固定。
- 首次打开页面时不会请求 60 秒视频文件。
- 点击视频封面后才开始请求并播放网页视频。
- 开启系统“减少动态效果”后，首屏保持静态。
- 桌面和手机布局均无溢出、跳动或不可读文字。
- 构建、代码检查、SEO 检查和差异检查全部通过。

## 验证

- 为轮播顺序、减少动态效果和视频延迟加载行为添加自动化测试。
- 运行 `npm run build`。
- 运行 `npm run lint`。
- 运行 `node scripts/seo-check.mjs`。
- 运行 `git diff --check`。
- 在浏览器中检查桌面和移动端首屏、动画切换、减少动态效果以及视频点击加载。

## 非目标

- 不把 60 秒视频设置为首屏自动播放背景。
- 不添加重量级轮播、动画或视频播放器依赖。
- 不改写首页文案、导航结构或 SEO 架构。
- 不提交 `output/` 或 `.superpowers/` 预览文件。
